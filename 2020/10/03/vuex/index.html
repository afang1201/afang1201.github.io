Vuex

官网：https://vuex.vuejs.org/zh/

1.介绍

Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

2.Vuex能够干什么

可以试想这样的场景, 比如一个Vue的根实例下面有一个根组件名为App.vue, 它下面有两个子组件A.vue和B.vue, App.vue想要与A.vue或者B.vue通讯可以通过父子通讯props传值的方式, 但是如果A.vue和B.vue之间的通讯就很麻烦了, 他们需要共有的父组件通过自定义事件进行实现, A组件想要和B组件通讯往往是这样的:

1.在Vue 的 prototype上挂载了一个Event属性，该Event 属性指向一个 Vue 的实例

2.A组件说: “报告爸爸, 能否帮我托个信给弟弟B” =>通过Event 触发一个事件给App

2.App爸爸说: “没问题包在我身上, 我需要监听A组件的触发的时间, 然后分发这个事件给B组件”

3.弟弟B说: “信息已收到”, 我只需要借助声明周期钩子来监听App爸爸分发的事件

具体实现代码如下

//main.js挂载Event属性
Vue.prototype.Event=new Vue();
//组件A在函数方法中通过Event属性触发自定义事件sendA
methods:{
    send(){
        this.Event.$emit('sendA','弟弟吃了吗')
    }
},
 //组件B借助声明周期钩子监听父组件派发的自定义事件sendA 
    mounted(){
        this.Event.$on("sendA",(e)=>{
            console.log(e);
        })
    },

当然这只是两个兄弟之间的简单传递，如果涉及到更多那么之间的逻辑关系就会更加复杂，那有没有什么好的解决方式呢，这时候Vuex的作用就体现了出来，下面是我对Vuex核心概念的一些理解。

3.Vuex核心概念

1.State

state就是Vuex中的公共的状态, 可以将state看作是所有组件的data, 用于保存所有组件的公共数据.

这里的话借助一个demo进行展示

//App.vue设置两个组件A，B
<template>
  <div>
    <v-a></v-a>
    <v-b></v-b>
  </div>
</template>

//A组件代码块
<template>
  <div class="box">
      <h1>this is A</h1>
      <p>name:{{$store.state.name}}</p>
      <p>age:{{$store.state.age}}</p>
  </div>
</template>

//B组件代码块
<template>
  <div class="box">
      <h1>this is B</h1>
      <p>name:{{$store.state.name}}</p>
      <p>age:{{$store.state.age}}</p>
  </div>
</template>
//main.js设置Vuex
export default new Vuex.Store({
    //状态 数据
    state: {
        name: "妲己",
        age: 20
    }
})

2.Getters

可以将getters属性理解为所有组件的computed属性, 也就是计算属性. vuex的官方文档也是说到可以将getter理解为store的计算属性, getters的返回值会根据它的依赖被缓存起来，且只有当它的依赖值发生了改变才会被重新计算。

可以为vuex添加getters代码段

 //main.js设置Vuex
export default new Vuex.Store({
    //状态 数据
    state: {
        name: "妲己",
        age: 20
    }，
    //相当于计算属性，目的为了导出数据给组件使用
    getters:{
        name(state){
        return state.name
        },
        age(state){
            return state.age*2
        }
	}
})

更改AB组件代码块为

//A组件代码块
<template>
  <div class="box">
      <h1>this is A</h1>
		//A组件不作修改作为参照
      <p>name:{{$store.state.name}}</p>
      <p>age:{{$store.state.age}}</p>
  </div>
</template>
//B组件代码块
<template>
  <div class="box">
      <h1>this is B</h1>
		//$store.state.name改为了$store.getters.name
      <p>name:{{$store.getters.name}}</p>
      <p>age:{{$store.getters.age}}</p>
  </div>
</template>

此时B组件的age变为了A组件的2倍

3. mutations

可以将mutaions理解为store中的methods, mutations对象中保存着更改数据的回调函数,该函数名官方规定叫type, 第一个参数是state, 第二参数是payload, 也就是自定义的参数.

下面,我在main.js中添加mutations属性,其中changeName这个回调函数用于name的改变

//main.js设置Vuex
export default new Vuex.Store({
    //状态 数据
    state: {
        name: "妲己",
        age: 20
    }，
    //相当于计算属性，目的为了导出数据给组件使用
    getters:{
        name(state){
        return state.name
        },
        age(state){
            return state.age*2
        }
	}，
     //直接修改state
    mutations: {
        changeName(state, name) {
            state.name = name
        }
    },
})

在A组件中添加一个按钮,为其添加一个点击事件, 给点击事件触发changeName方法

//A组件代码块
<template>
  <div class="box">
      <h1>this is A</h1>
      <p>name:{{$store.state.name}}</p>
      <p>age:{{$store.state.age}}</p>
  	  <button @click="$store.commit('changeName','貂蝉')">貂蝉</button>
  </div>
</template>

4.Actions

actions 类似于 mutations，不同在于：

  • actions提交的是mutations而不是直接变更状态
  • actions中可以包含异步操作, mutations中绝对不允许出现异步
  • actions中的回调函数的第一个参数是context, 是一个与store实例具有相同属性和方法的对象
  • 此时,我们在store中添加actions属性, 其中changeNameAsync采用setTimeout来模拟异步操作,延迟2s执行 该方法用于异步改变我们刚才在mutaions中定义的name
//main.js设置Vuex
export default new Vuex.Store({
    //状态 数据
    state: {
        name: "妲己",
        age: 20
    }，
    //相当于计算属性，目的为了导出数据给组件使用
    getters:{
        name(state){
        return state.name
        },
        age(state){
            return state.age*2
        }
	}，
     //直接修改state
    mutations: {
        changeName(state, name) {
            state.name = name
        }
    },
     // 接收组件的触发的动作，可以处理异步，但是不能修改state，最后会commit到mutations
    actions:{
        //context 就是你的仓库本身，是个只读的
        changeNameAsync(context,name){
            setTimeout( () => {
                context.commit( 'changeName', name ); //context提交
            }, 2000)
        }
    }
})

在A组件中添加一个按钮,为其添加一个点击事件, 给点击事件触发changeNameAsync方法

//A组件代码块
<template>
  <div class="box">
      <h1>this is A</h1>
      <p>name:{{$store.state.name}}</p>
      <p>age:{{$store.state.age}}</p>
  	  <button @click="$store.commit('changeName','貂蝉')">貂蝉</button>
  	   <button @click="$store.dispatch('changeNameAsync','王昭君')">action 王昭君</button>
  </div>
</template>

点击action按钮后两秒以后的页面效果

5.Modules

由于使用单一状态树，应用的所有状态会集中到一个比较大的对象。当应用变得非常复杂时，store 对象就有可能变得相当臃肿。为了解决以上问题，Vuex 允许我们将 store 分割成模块（module）。每个模块拥有自己的 state、mutation、action、getter、甚至是嵌套子模块——从上至下进行同样方式的分割

const moduleA = {
  state: { ... },
  mutations: { ... },
  actions: { ... },
  getters: { ... }
}

const moduleB = {
  state: { ... },
  mutations: { ... },
  actions: { ... }
}

const store = new Vuex.Store({
  modules: {
    a: moduleA,
    b: moduleB
  }
})

store.state.a // -> moduleA 的状态
store.state.b // -> moduleB 的状态

小结：

1.mutations VS actions

1.mutations 只能处理同步，action可以处理异步
2.触发mutations通过commit,触发actions通过dispatch
3.mutations用来修改state,actions用来接收组件派发的动作
