组件通信

1.父传子

父传子：父组件将要传递的数据通过自定义属性传递给子组件，子组件通过props接收

    <!--父组件-->
    <template>
      <div>
        <v-child :a='1'></v-child>
      </div>
    </template>

    <script>
    import  vChild from './child'
    export default {
      components:{
        vChild
      }
    }
    </script>

    <style>

</style>
//子组件
<template>
  <div class="box">
      <h3>这是子组件</h3>
      <p>父组件传递给我的数据是：{{a}}</p>
  </div>
</template>

<script>
export default {
    props:["a"]
}
</script>

<style>

</style>

扩展：

  • 1. 在子组件js使用父组件传递过来的数据，用this.a
2.对于一个组件来说，模板中可以使用的变量只能是自己的props data computed ，能直接修改的只有data和computed.
3. 父组件传递过来的数据，子组件不允许直接修改，按标准来做，应该使用子传父，实现子组件修改父组件的数据
4.如果子组件直接修改父组件传递过来的值，子变，父不变，还报错
5.父变，子变；子变，父变.可以传递json数据解决。

2.子传父

目的：子组件要修改父组件的值

在父组件绑定自定义事件，子组件通过$emit()触发自定义事件，携带的参数，在event上

<!--父组件-->
<v-child2  @change="changeName($event)"></v-child2>
//子组件
export default {
    methods:{
        change(name){
            //通知父组件，把父组件的name改成功 name
            this.$emit("change",name)
        }
    }
}

4.通过$children与$parent实现父子通信

1.子组件通过$parent获得父组件传递的值
//子组件
<template>
  <div class="box">
      <h3>这是子组件</h3>
      <button @click="getInfo">获得父组件传递信息</button>
      <p>父组件传递给我的数据是：{{msg}}</p>
  </div>
</template>

<script>
export default {
  // props:["a"]，
  data(){
    return{
      msg:''
    }
  },
  methods:{
    getInfo(){
     this.msg=this.$parent.msg
    }
  }
   
}
</script>
//父组件
<template>
  <div>
    <v-child></v-child>
  </div>
</template>

<script>
import  vChild from './child'
export default {
  data(){
    return{
      msg:'这是父组件的信息'
    }
  },
  components:{
    vChild
  }
}
</script>

2.父组件通过$children获得子组件传递的值
//子组件
<template>
  <div class="box">
      <h3>这是子组件</h3>
      <p>{{msg}}</p>
  </div>
</template>

<script>
export default {
  // props:["a"]，
  data(){
    return{
      msg:'这是子组件的信息'
    }
  }
}
//父组件
<template>
  <div>
    <button @click="editChild">改变</button>
    <v-child></v-child>
  </div>
</template>

<script>
import vChild from './child'
export default {
  components: {
    vChild,
  },
  methods:{
    editChild(){
      console.log(this.$children);
      //注意这里要加下标
      this.$children[0].msg = '已改变'
    }
  }
}
</script>

在这里解释下通过$children改变子组件的msg值时为什么要加下标，很多博客关于$children的用法教程都是直接在后面调变量名用，这种用法是错误的，不但不会出效果还会报错！！在这里贴一张$children的打印信息大家就明白了。

可以发现打印出来的$children并不像$parent一样是一个对象而是一个数组，因为子组件的数量是不确定的所以$childeren也并不能保证顺序。如果涉及到多个子组件的取值建议使用自定义事件来获取子组件传值
