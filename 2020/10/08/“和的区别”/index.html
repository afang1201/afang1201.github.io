“$route和$router的区别”

学习笔记
vue
发布日期:   2020-10-08
更新日期:   2020-10-09
文章字数:   531
阅读时长:   2 分

一、使用场景

首先说一下$router和$route的使用场景吧

1.$router

router为VueRouter的实例，相当于一个全局的路由器对象，里面含有很多属性和子对象。主要用于页面的跳转。

关于$router最经常用的两个方法分别是push和replace虽然两者都能实现页面的跳转功能，但是还是存在一些细微的差异。

其中$router.push({path:’home’});本质是向history栈中添加一个路由，也就是添加一个history记录，我们可以通过浏览器后退返回上个路由页面。

而$router.replace({path:’home’});则是替换路由，没有历史记录，后退将会离开该页面进入该页面的上级页面。

2.$route

route是一个跳转的路由对象，每一个路由都会有一个route对象，是一个局部的对象，可以获取对应的name,path,params,query等

  • $route.path
    字符串，等于当前路由对象的路径，会被解析为绝对路径，如 "/home/news" 。

  • $route.params
    对象，包含路由中的动态片段和全匹配片段的键值对

    可以通过代码段来进行理解

    //router >> index.js

{
//意思就是往 Vue.$route.params里储存属性"name"和"age",作为键名key,
        path: '/index/:name/:age',
        name: 'index',
        component: index

    }
//App.vue
//点击就跳转到index,并同时传参到Vue.$route.params
<router-link to='/index/a/18'></router-link>
          

//index.vue
<template>
//获得属性age的值
    <div>{{this.$route.params.age}}</div>
</template>

同时还可以同过编程路由的方式进行传参

this.$router.push(
    {
		path:'/index',
		name:'index',
		params:{
			name:'xx',
			age:'xx'
		}
		
    }
)

  • $route.query
    对象，包含路由中查询参数的键值对。例如，对于 /home/news/detail/01?favorite=yes ，会得到$route.query.favorite == 'yes' 。

  • $route.router
    路由规则所属的路由器（以及其所属的组件）。

  • $route.matched
    数组，包含当前匹配的路径中所包含的所有片段所对应的配置参数对象。

  • $route.name
    当前路径的名字，如果没有使用具名路径，则名字为空。
